## 首次登入

1. 啟動系統後，使用瀏覽器開啟 `http://localhost:3000`
2. 使用預設帳號登入：
   - 帳號: `admin`
   - 密碼: `admin`
3. **重要**: 登入後請到「權限管理」更改密碼

## 儀表板

儀表板提供系統的總覽資訊：

- **庫存品項數**: 目前所有商品的品項數
- **庫存成本**: 所有庫存的總成本
- **預計利潤**: 以售價減成本計算的預計利潤
- **已完成訂單**: 已完成的訂單數量
- **庫存分類圓餅圖**: 各分類的數量佔比
- **成本 vs 售價長條圖**: 各分類的成本與售價對比
- **低庫存警示**: 庫存低於最低數量的商品

## 庫存管理

### 新增商品

1. 點擊右上角 **+ 新增商品**
2. 填寫以下欄位：
   - **分類**: CPU/顯示卡/主機板/記憶體/硬碟/散熱/電源供應器/機殼/周邊
   - **品名**: 商品名稱（必填）
   - **品牌**: 例如 AMD, Intel, NVIDIA
   - **型號**: 例如 R5-7600X
   - **規格**: 例如 6C/12T 4.7GHz
   - **成本**: 進貨成本
   - **售價**: 預計售價
   - **數量**: 目前庫存量
   - **最低庫存**: 低於此數量會顯示警示
3. 點擊 **新增** 儲存

### 篩選與搜尋

- 使用分類下拉選單篩選特定分類
- 搜尋框可搜尋品名、品牌、型號

### 庫存異動

系統會自動記錄庫存數量的變動，包含初始庫存和手動調整。

## 客戶估價

### 建立估價單

1. 點擊 **+ 新增估價單**
2. 填寫客戶名稱
3. 在項目明細中加入零件：
   - 選擇分類（可選擇相同分類多次，例如多條記憶體、多顆硬碟）
   - 填寫品名、規格、成本、售價、數量
   - 點擊 **+ 新增項目** 加入更多零件
4. 設定**服務費**（組裝費、安裝費等）
5. 填寫訂金金額
6. 系統自動計算：
   - **總成本** = 所有項目成本 × 數量之和
   - **總售價** = 所有項目售價 × 數量之和 + 服務費
   - **毛利** = 總售價 - 總成本
   - **毛利率** = 毛利 / 總售價 × 100%

### 訂單狀態流程

```
草稿 → 已付訂金 → 尚未結單 → 已完成
                              ↘ 已取消
```

### 上傳圖片

1. 先儲存估價單
2. 編輯估價單時，可在底部上傳圖片
3. 支援多張圖片上傳（最大 10MB/張）
4. 可點擊圖片放大檢視或刪除

### 匯出估價單

點擊匯出按鈕（📥），系統會開啟新視窗顯示可列印的估價單，包含：
- 估價單編號、客戶名稱、日期
- 零件明細表
- 服務費（如有）
- 總計金額
- 訂金與尚需付款金額（如有訂金）

## 盤商報價

### 解析 Line 訊息

1. 填寫盤商名稱（例如：原價屋、欣亞）
2. 將盤商在 Line 傳來的報價訊息直接貼上
3. 點擊 **解析報價**

### 支援的格式

```
AMD R5-7600X $4,590          → 品名: AMD R5-7600X, 價格: 4,590
RTX 4060 Ti NT$12,800        → 品名: RTX 4060 Ti, 價格: 12,800
B660M：3290                   → 品名: B660M, 價格: 3,290
海韻 650W 2890元              → 品名: 海韻 650W, 價格: 2,890
NVMe 1TB 2090                → 品名: NVMe 1TB, 價格: 2,090
```

### 歷史報價查詢

右側面板可搜尋歷史報價紀錄，方便比價。

## 客戶管理

### 新增客戶

1. 點擊 **+ 新增客戶**
2. 填寫客戶資訊：
   - 姓名（必填）
   - 電話、Email
   - **來源**: YouTube / Instagram / Line / 門市 / 朋友介紹 / 其他
   - **地區**: 縣市 + 區域
3. 儲存

### 服務紀錄

點擊客戶名稱檢視詳情，可：
- 新增服務紀錄（組裝、維修、升級、諮詢等）
- 查看該客戶的所有估價單

### 台灣地圖

右側會顯示客戶的地區分佈圖：
- 圓點大小代表該地區客戶數量
- 滑鼠懸停可看到具體數字
- 下方有來源分佈圖表和地區排名

## YouTube 設定

### 取得 API Key

1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立專案或選擇既有專案
3. 啟用 **YouTube Data API v3**
4. 建立 API Key（憑證 → 建立憑證 → API 金鑰）

### 取得頻道 ID

1. 前往你的 YouTube 頻道首頁
2. 頻道 ID 為網址中 `UC` 開頭的字串
3. 或到 YouTube 進階設定中查看

### 設定步驟

1. 進入 YouTube 頁面
2. 貼上 API Key 和頻道 ID
3. 點擊 **儲存設定**
4. 點擊 **查詢訂閱數** 即可看到即時資料

## 權限管理

> 僅管理員帳號可存取此功能

### 新增使用者

1. 點擊 **+ 新增使用者**
2. 設定帳號、密碼、顯示名稱
3. 選擇角色：
   - **管理員**: 擁有全部權限
   - **使用者**: 依據勾選的頁面權限存取
4. 勾選允許存取的頁面

### 可控制的頁面

- 儀表板
- 庫存管理
- 客戶估價
- 盤商報價
- 客戶管理
- YouTube

### 停用帳號

可將使用者狀態設為「停用」，該帳號將無法登入。

## 內網部署

### 直接啟動

```bash
# 安裝
pnpm install

# 啟動（背景執行）
nohup node src/server.js > app.log 2>&1 &
```

### 使用 Docker

```bash
docker compose up -d --build
```

### 內網存取

系統啟動時會顯示內網 IP 位址。其他電腦使用該 IP + Port 存取即可：

```
http://192.168.1.100:3000
```

### 資料備份

```bash
# 備份資料庫
cp data/shop.db data/shop.db.backup

# 備份上傳的圖片
cp -r data/uploads/ data/uploads.backup/
```

### 建議

- 定期備份 `data/` 資料夾
- 預設密碼請務必更改
- 如需多人同時使用，建議使用 Docker 部署確保穩定性
